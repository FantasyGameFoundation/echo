# Low-Fidelity Page Skeleton: Photographer Project Continuity App

## Purpose

本文件定义第一版移动端页面骨架和低保真区块，不涉及视觉风格，只说明：

- 页面上有哪些区块
- 每个区块承担什么职责
- 用户在手机端如何完成流转

## Global Layout

第一版建议统一采用：

- 顶部项目上下文栏
- 中央滚动主内容区
- 底部一级导航
- 必要时使用底部弹层或全屏编辑页

设计意图：

- 适应手机端小屏
- 减少复杂嵌套导航
- 把核心动作压缩到清晰的单任务流里

## Bottom Navigation

建议 4 个一级入口：

- `项目`
- `结构`
- `整理`
- `历程`

其中：

- `整理` 应作为最突出的主工作入口

## Page 1: 项目主页

### Layout Skeleton

顶部：
- 项目标题
- 当前阶段
- 编辑项目信息入口

首屏主卡：
- 主题陈述
- 项目简介摘要

进度区：
- 结构进度卡
- 元素完成度卡
- 待整理照片数量卡

最近动态区：
- 最近导入照片
- 最近创作想法
- 最近完成元素

底部固定主按钮：
- 继续整理

### Core Questions This Page Answers

- 这个项目当前在做什么？
- 还差多少元素没完成？
- 我下一步最该去哪里？

### Exit Paths

- 去结构页
- 去整理页
- 去创作时间线

## Page 2: 结构页

### Recommended Layout

顶部：
- 页面标题
- 新建按钮

上方分段切换：
- `结构`
- `元素`

主内容区：
- 当前分段下的列表

详情呈现方式：
- 点击进入详情页
- 或使用底部弹层查看详情

### Structure Segment

列表项内容：
- 节点名称
- 简短说明
- 关联元素数量
- 关联照片数量

可执行动作：
- 新建结构节点
- 调整顺序
- 查看节点详情

### Element Segment

列表项内容：
- 元素名称
- 当前状态
- 支撑照片数量
- 所属结构摘要

可执行动作：
- 新建元素
- 标记完成
- 查看元素详情
- 打开支撑照片

### Detail View

当查看结构节点详情时：
- 节点说明
- 相关元素
- 相关照片摘要

当查看元素详情时：
- 元素说明
- 支撑照片
- 当前状态
- 消项入口

### Core Questions This Page Answers

- 这本作品的结构骨架是什么？
- 每个结构包含哪些内容材料？
- 哪些元素还没被满足？

## Page 3: 整理页

这是 MVP 的核心页面。

### Recommended Layout

顶部：
- 当前项目名
- 待整理数量

外拍信息流区：
- 最近信息流节点列表
- 每个节点显示时间、地点、文字摘要、临时照片数量、正式照片数量
- 新建信息流按钮

主图片区：
- 当前选中照片大图或预览
- 左右切换上一张 / 下一张

下方信息区：
- 这张照片的简短备注
- 编辑备注入口

下方操作区采用折叠卡或分步区块：

1. 结构挂接
2. 元素挂接
3. 关系属性
4. 创作思路
5. 消项动作

底部固定操作栏：
- 保存
- 下一张

### Capture Feed Section

内容：
- 信息流节点时间
- 信息流节点地点
- 节点文字摘要
- 节点内的临时照片与正式导入照片摘要

交互建议：
- 10 分钟内的连续记录自动并入一个节点
- 用户可点开节点查看内部内容
- 用户可把正式导入照片重新关联到其他节点

### Structure Section

内容：
- 当前已选结构
- 选择结构按钮

选择方式建议：
- 打开全屏选择页
- 或底部弹层单选

### Element Section

内容：
- 已关联元素列表
- 添加元素按钮
- 搜索元素入口
- 快速新建元素入口

选择方式建议：
- 进入轻量选择页
- 支持多选

### Relation Attribute Section

内容：
- 呼应
- 对比
- 重复
- 转折

交互建议：
- 使用标签式选择
- 不做复杂图谱编辑

### Idea Section

内容：
- 当前照片相关想法
- 与项目整体相关的补充思路

### Completion Section

内容：
- 允许对本次已满足的元素执行消项
- 应清楚显示“本次照片支撑了哪些元素”

### Core Questions This Page Answers

- 这张照片属于项目的哪个部分？
- 它支撑了哪些元素？
- 它和项目已有材料是什么关系？
- 它是否让某个元素可以被完成？

### Critical Design Rule

用户必须能在手机端较少步骤内完成一次整理。  
不能让一次整理退化成多页面来回跳转。
同时必须让“信息流节点 -> 正式照片 -> 结构/元素”的链条是可见的。

## Page 4: 创作时间线

### Recommended Layout

顶部：
- 时间范围筛选
- 事件类型筛选

主内容区：
- 时间线事件列表

每个事件卡片显示：
- 时间
- 事件类型
- 简短描述
- 关联对象摘要

事件详情打开方式：
- 独立详情页
- 或底部弹层

### Event Types

- 照片导入
- 元素完成
- 新增创作想法
- 结构调整

### Core Questions This Page Answers

- 这个项目是怎么慢慢长出来的？
- 哪些想法是什么时候进入项目的？
- 哪些元素是在什么时候真正成立的？

### Design Constraint

它必须像“创作历程回看”，不能像普通操作日志页。

## Cross-Page Interaction Skeleton

### From 项目主页

- 点击“继续整理” -> 整理页，定位到未整理照片
- 点击结构进度卡 -> 结构页
- 点击最近动态 -> 创作时间线

### From 结构页

- 点击某个元素 -> 元素详情
- 点击“查看支撑照片” -> 整理页或照片上下文
- 点击“消项” -> 当前页轻量完成，或跳转整理页确认

### From 整理页

- 保存后 -> 当前页继续下一张
- 点击结构名 / 元素名 -> 可进入详情页查看上下文
- 点击时间信息 -> 可打开对应时间线事件

### From 创作时间线

- 点击事件 -> 打开对应照片、元素或想法
- 点击“回到整理” -> 整理页

## Low-Fi Priority

如果只先画 3 张移动端线框图，优先顺序应是：

1. 整理页
2. 结构页
3. 项目主页

时间线第四张再画。

原因：

- 整理页决定 MVP 是否成立
- 结构页决定对象模型是否清晰
- 项目主页决定项目上下文是否好理解

## Prototype Questions to Validate

画完低保真后，应优先验证：

1. 用户能不能在手机端一眼分清结构和元素
2. 用户能不能顺手完成单张照片的挂接和消项
3. 用户会不会觉得整理步骤过多
4. 用户能不能从项目主页迅速回到核心任务
5. 时间线是否像“作品形成过程”，而不是普通日志
